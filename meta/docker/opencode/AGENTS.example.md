# Global AGENTS

- `cowork` is the preferred local coordination entrypoint for managed knowledge repositories.
- When a repository looks like a managed knowledge repo, check whether `cowork` is available before inventing an ad-hoc path.
- Treat `cowork clone preview`, `cowork clone contribute`, and `cowork clone resource` as guide-only commands.
- If the local environment already has the target repository and `cowork`, prefer using `cowork` to understand repository boundary and collaboration flow.
